# 🏗️ نظام طلبات المطاعم

# 04 - بنية النظام (System Architecture)

---

# مقدمة

توضح هذه الوثيقة البنية العامة للنظام، وكيفية تواصل جميع مكونات المشروع مع بعضها البعض.

كما تساعد جميع أعضاء الفريق على فهم مكان كل جزء داخل النظام، ومسؤولية كل تقنية مستخدمة.

---

# معمارية النظام

يعتمد المشروع على معمارية Client - Server Architecture.

أي أن النظام يتكون من:

- واجهة أمامية (Frontend)
- خادم (Backend)
- قاعدة بيانات (Database)

ويتواصل كل جزء مع الآخر من خلال REST API.

---

# مكونات النظام

## أولاً: واجهة المستخدم (Frontend)

التقنيات المستخدمة

- React
- Vite
- Tailwind CSS

المسؤوليات

- عرض صفحات الموقع.
- عرض المنيو.
- عرض تفاصيل الوجبات.
- إدارة سلة المشتريات.
- إرسال طلبات إلى الـ API.
- استقبال البيانات وعرضها للمستخدم.

---

## ثانياً: الخادم (Backend)

التقنيات المستخدمة

- ASP.NET Core Web API

المسؤوليات

- استقبال الطلبات من الـ Frontend.
- تنفيذ منطق العمل (Business Logic).
- التعامل مع قاعدة البيانات.
- إنشاء الطلبات.
- تجهيز رسالة WhatsApp.
- إعادة البيانات بصيغة JSON.

---

## ثالثاً: قاعدة البيانات (Database)

التقنية المستخدمة

SQL Server

المسؤوليات

- تخزين بيانات الوجبات.
- تخزين التصنيفات.
- تخزين بيانات العملاء.
- تخزين الطلبات.
- تخزين تفاصيل كل طلب.

---


# طريقة التواصل بين Frontend و Backend

يعتمد المشروع على REST API.

يقوم الـ Frontend بإرسال طلبات HTTP إلى الـ Backend.

ويقوم الـ Backend بإرجاع البيانات بصيغة JSON.

مثال:

```
GET /api/meals
```

يقوم بإرجاع جميع الوجبات.

---

```
GET /api/categories
```

يقوم بإرجاع جميع التصنيفات.

---

```
POST /api/orders
```

يقوم بإنشاء طلب جديد.

# مكان الذكاء الاصطناعي داخل النظام

لن يكون الذكاء الاصطناعي جزءًا من الإصدار الأول.

سيتم إضافته في إصدار مستقل.

سيعتمد على نفس قاعدة البيانات ونفس الـ API.

وسيستخدم البيانات الموجودة لتقديم خدمات مثل:

- اقتراح وجبات مشابهة.
- البحث الذكي.
- اقتراح وجبات حسب الميزانية.
- عرض أكثر الوجبات طلبًا.

---

# دعم تطبيق Flutter

إذا تم تطوير تطبيق Flutter مستقبلًا، فلن يحتاج إلى Backend جديد.

سيستخدم نفس الـ API ونفس قاعدة البيانات.

وسيكون الشكل كالتالي:

```
React Website
        │
        │
Flutter App
        │
        ▼
ASP.NET Core Web API
        │
        ▼
SQL Server
```

وبذلك يستطيع أكثر من تطبيق استخدام نفس البيانات.

---

# المبادئ المعتمدة في المشروع

يعتمد المشروع على المبادئ التالية:

- فصل المسؤوليات بين Frontend و Backend.
- استخدام REST API للتواصل.
- عدم وصول Frontend مباشرة إلى قاعدة البيانات.
- جميع البيانات تمر عبر Backend.
- الحفاظ على بنية بسيطة وقابلة للتطوير.

---

# ملاحظات

- جميع التعديلات على الـ API يجب أن يتم الاتفاق عليها بين Frontend و Backend.
- أي تقنية جديدة يجب توثيقها قبل استخدامها.
- سيتم تحديث هذه الوثيقة عند إضافة خدمات جديدة أو تغيير بنية النظام.
