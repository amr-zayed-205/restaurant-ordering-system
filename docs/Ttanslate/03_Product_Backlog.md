# 📌 نظام طلبات المطاعم

# 03 - قائمة المهام والخصائص (Product Backlog)

---

# مقدمة

تحتوي هذه الوثيقة على جميع خصائص المشروع والمهام الرئيسية المتوقع تنفيذها.

سيتم لاحقًا تحويل كل عنصر من هذه القائمة إلى GitHub Issue أو Task داخل GitHub Projects.

لا يشترط تنفيذ جميع المهام في إصدار واحد، وإنما يتم توزيعها على عدة إصدارات (Versions).

---

# الوحدة الأولى: الصفحة الرئيسية (Home)

## الخصائص

- إنشاء الصفحة الرئيسية.
- عرض شعار المطعم.
- عرض اسم المطعم.
- عرض نبذة مختصرة عن المطعم.
- زر الانتقال إلى المنيو.
- شريط التنقل (Navigation Bar).
- تذييل الصفحة (Footer).

---

# الوحدة الثانية: المنيو (Menu)

## الخصائص

- عرض جميع الوجبات.
- عرض جميع التصنيفات.
- البحث عن الوجبات.
- تصفية الوجبات حسب التصنيف.
- عرض الوجبات بطريقة Grid.

---

## بطاقة الوجبة (Meal Card)

يجب أن تحتوي البطاقة على:

- صورة الوجبة.
- اسم الوجبة.
- وصف مختصر.
- السعر.
- زر "إضافة إلى السلة".

---

# الوحدة الثالثة: تفاصيل الوجبة (Meal Details)

## الخصائص

- عرض صورة كبيرة للوجبة.
- عرض اسم الوجبة.
- عرض الوصف الكامل.
- عرض السعر.
- عرض المكونات (اختياري).
- زر إضافة إلى السلة.

---

# الوحدة الرابعة: سلة المشتريات (Shopping Cart)

## الخصائص

- عرض جميع المنتجات المضافة.
- زيادة الكمية.
- تقليل الكمية.
- حذف منتج.
- إفراغ السلة بالكامل.
- حساب إجمالي السعر.

---

# الوحدة الخامسة: إتمام الطلب (Checkout)

## بيانات العميل

البيانات المطلوبة:

- الاسم.
- رقم الهاتف.
- العنوان.

البيانات الاختيارية:

- ملاحظات.

---

# الوحدة السادسة: الطلب (Order)

## الخصائص

- إنشاء طلب جديد.
- إنشاء ملخص الطلب.
- تجهيز رسالة WhatsApp.
- فتح WhatsApp.
- إرسال الطلب.

---

# الوحدة السابعة: Backend

## واجهات البرمجة (API)

- API الخاصة بالوجبات.
- API الخاصة بالتصنيفات.
- API الخاصة بالطلبات.

---

## منطق العمل (Business Logic)

- التحقق من صحة بيانات العميل.
- حساب إجمالي الطلب.
- إنشاء الطلب.
- تجهيز رسالة WhatsApp.

---

# الوحدة الثامنة: قاعدة البيانات

الجداول الأساسية:

- Categories
- Meals
- Customers
- Orders
- OrderItems

---

# الوحدة التاسعة: الذكاء الاصطناعي (مستقبلاً)

خصائص مقترحة:

- اقتراح وجبات مشابهة.
- اقتراح وجبات حسب الميزانية.
- البحث الذكي.
- أكثر الوجبات طلبًا.
- اقتراح وجبات يتم طلبها معًا.

---

# الوحدة العاشرة: الاختبارات (Testing)

تشمل:

- اختبار الواجهة.
- اختبار الـ API.
- اختبار قاعدة البيانات.
- اختبار إرسال الطلب إلى WhatsApp.
- اختبار التصميم على مختلف الأجهزة.

---

# خطة الإصدارات

## الإصدار الأول

- تصميم الواجهة.
- إنشاء مشروع React.
- إنشاء مشروع Backend.
- إعداد ملفات JSON.
- عرض البيانات باستخدام JSON.

---

## الإصدار الثاني

- إنشاء قاعدة البيانات.
- إنشاء API.
- ربط React بالـ API.

---

## الإصدار الثالث

- إنشاء سلة المشتريات.
- صفحة إتمام الطلب.
- إرسال الطلب عبر WhatsApp.

---

## الإصدار الرابع

- إضافة مميزات الذكاء الاصطناعي.

---

# ملاحظات

- سيتم تحويل جميع العناصر السابقة إلى مهام (Tasks) داخل GitHub Projects.
- يمكن إضافة أو حذف أي مهمة حسب احتياجات المشروع.
- تعتبر هذه الوثيقة المرجع الرئيسي لتخطيط العمل داخل الفريق.
