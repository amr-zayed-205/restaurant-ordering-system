# 🗄️ نظام طلبات المطاعم

# 06 - تصميم قاعدة البيانات

---

# مقدمة

توضح هذه الوثيقة تصميم قاعدة البيانات المستخدمة في نظام طلبات المطاعم.

الهدف من هذا التصميم هو تنظيم البيانات بطريقة بسيطة، قابلة للتطوير، وسهلة الصيانة.

قاعدة البيانات المستخدمة في المشروع هي:

**SQL Server**

---

# الكيانات الأساسية (Entities)

يتكون الإصدار الأول من المشروع من خمس كيانات رئيسية:

- Categories (التصنيفات)
- Meals (الوجبات)
- Customers (العملاء)
- Orders (الطلبات)
- OrderItems (عناصر الطلب)

---

# الجداول

---

## جدول التصنيفات (Categories)

### الغرض

يحتوي على جميع تصنيفات الوجبات.

### الحقول

| الحقل | النوع | الوصف |
|--------|--------|--------|
| Id | INT | رقم التصنيف |
| Name | NVARCHAR(100) | اسم التصنيف |

---

## جدول الوجبات (Meals)

### الغرض

يحتوي على جميع الوجبات الموجودة في المنيو.

### الحقول

| الحقل | النوع | الوصف |
|--------|--------|--------|
| Id | INT | رقم الوجبة |
| Name | NVARCHAR(150) | اسم الوجبة |
| Description | NVARCHAR(MAX) | وصف الوجبة |
| Price | DECIMAL | السعر |
| ImageUrl | NVARCHAR(255) | رابط الصورة |
| CategoryId | INT | رقم التصنيف |

---

## جدول العملاء (Customers)

### الغرض

تخزين بيانات العملاء.

### الحقول

| الحقل | النوع | الوصف |
|--------|--------|--------|
| Id | INT | رقم العميل |
| FullName | NVARCHAR(150) | الاسم |
| PhoneNumber | NVARCHAR(20) | رقم الهاتف |
| Address | NVARCHAR(255) | العنوان |

---

## جدول الطلبات (Orders)

### الغرض

تخزين الطلبات التي ينشئها العملاء.

### الحقول

| الحقل | النوع | الوصف |
|--------|--------|--------|
| Id | INT | رقم الطلب |
| CustomerId | INT | رقم العميل |
| OrderDate | DATETIME | تاريخ الطلب |
| TotalPrice | DECIMAL | إجمالي السعر |
| Notes | NVARCHAR(MAX) | ملاحظات |

---

## جدول عناصر الطلب (OrderItems)

### الغرض

تحديد الوجبات الموجودة داخل كل طلب.

### الحقول

| الحقل | النوع | الوصف |
|--------|--------|--------|
| Id | INT | رقم العنصر |
| OrderId | INT | رقم الطلب |
| MealId | INT | رقم الوجبة |
| Quantity | INT | الكمية |
| UnitPrice | DECIMAL | سعر الوحدة وقت الطلب |

---

# ملخص العلاقات

| العلاقة | النوع |
|----------|--------|
| Category → Meals | One To Many |
| Customer → Orders | One To Many |
| Order → OrderItems | One To Many |
| Meal → OrderItems | One To Many |

# قواعد التصميم

- لكل جدول مفتاح أساسي (Primary Key).
- جميع العلاقات تستخدم مفاتيح خارجية (Foreign Keys).
- يمنع تكرار البيانات غير الضروري.
- يتم الاحتفاظ بسعر الوجبة داخل OrderItems حتى إذا تغير سعرها لاحقًا.
