# 📋 Restaurant Ordering System

# Software Requirements Specification (SRS)

---

# 1. Functional Requirements

The following requirements describe the core functionality of the system.

---

# Epic 1 — Browse Menu

## User Story 1

As a customer,

I want to browse all available meals,

So that I can choose what I want to order.

### Acceptance Criteria

- Display all meals.
- Display meal image.
- Display meal name.
- Display meal price.
- Display meal category.

---

## User Story 2

As a customer,

I want to search for meals,

So that I can quickly find the food I want.

### Acceptance Criteria

- Search by meal name.
- Search updates instantly.

---

## User Story 3

As a customer,

I want to filter meals by category,

So that I can browse similar meals.

### Acceptance Criteria

- Categories displayed.
- Selecting category filters meals.

---

# Epic 2 — Meal Details

## User Story 4

As a customer,

I want to view meal details,

So that I know what I am ordering.

### Acceptance Criteria

- Meal image
- Description
- Price
- Ingredients (optional)

---

# Epic 3 — Shopping Cart

## User Story 5

As a customer,

I want to add meals to my cart,

So that I can order multiple meals.

### Acceptance Criteria

- Add meal.
- Increase quantity.
- Decrease quantity.
- Remove meal.

---

## User Story 6

As a customer,

I want to see my total price,

So that I know how much I will pay.

### Acceptance Criteria

- Calculate total automatically.
- Update when quantity changes.

---

# Epic 4 — Checkout

## User Story 7

As a customer,

I want to enter my personal information,

So that the restaurant can contact me.

### Acceptance Criteria

Required Fields:

- Name
- Phone Number
- Address

Optional:

- Notes

---

# Epic 5 — Place Order

## User Story 8

As a customer,

I want my order to be sent to the restaurant through WhatsApp,

So that I can complete my order easily.

### Acceptance Criteria

- Generate formatted message.
- Open WhatsApp.
- Include customer information.
- Include ordered meals.
- Include total price.

---

# 2. Non-Functional Requirements

The system should:

- Be responsive.
- Load quickly.
- Be easy to use.
- Support mobile devices.
- Have a clean UI.
- Follow modern web standards.

---

# 3. Out of Scope

The following features are NOT included in this version.

- Login
- Admin Dashboard
- Payment Gateway
- Coupons
- Notifications
- Inventory Management
- Employee Accounts
- AI Chatbot

---

# 4. Future Enhancements

Possible future improvements:

- AI Meal Recommendation
- Smart Search
- Order Tracking
- Customer Accounts
- Online Payments
- Restaurant Dashboard
